## 问题与阻塞

### Progress API 403（已解决）
- **描述**：Progress API 返回 403 Forbidden
- **影响**：前端无法获取进度数据，控制台报错
- **根因**：Progress 相关代码在清理 SaaS 二期时被误删
- **解决**：恢复 Progress 实体 + Repository + Service + Controller
- **状态**：🟢 已解决

### 字段名冲突（已解决）
- **描述**：服务启动失败，报错 `DuplicateMappingException`
- **影响**：后端服务无法启动
- **根因**：planId 和 taskId 字段映射歧义（JPA 自动推断列名）
- **解决**：添加 `@Column(name = "plan_id")` 和 `@Column(name = "task_id")` 明确列名
- **状态**：🟢 已解决

### 服务启动失败（已解决）
- **描述**：服务启动后健康检查返回 502
- **影响**：所有 API 无法访问
- **根因**：字段名冲突导致 Spring 上下文初始化失败
- **解决**：修复实体后重新编译部署
- **状态**：🟢 已解决

## 技术决策

### 决策 1：Progress 实体字段命名
- **背景**：JPA 自动推断列名导致冲突
- **选项**：
  - A. 使用默认命名（planId → planId）
  - B. 使用 `@Column` 明确指定（planId → plan_id）
- **决定**：选项 B
- **理由**：数据库表已存在，使用下划线命名符合数据库规范

### 决策 2：发布验证流程
- **背景**：多次发布后出现问题，需要标准化验证流程
- **选项**：
  - A. 仅 career-worker 自测
  - B. career-worker + qa-tester 双重验证
- **决定**：选项 B
- **理由**：独立验证可以发现遗漏问题，提高发布质量
