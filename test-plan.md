## 🧪 任务 3.1: 后端 API 测试

### API 列表
| API | 方法 | 端点 | 测试重点 |
|-----|------|------|----------|
| 修改密码 | PUT | `/api/v1/user/password` | 身份验证、密码强度、旧密码验证 |
| 查询用户资料 | GET | `/api/v1/user/{userId}/profile` | 权限控制（仅顾问）、数据完整性 |
| 任务确认 | POST | `/api/v1/tasks/{taskId}/confirm` | 幂等性、状态流转、权限验证 |
| 更新资料 | PUT | `/api/v1/user/profile` | 数据校验、必填字段、长度限制 |

### 测试类型
1. **单元测试** - 验证 Service 层逻辑
2. **集成测试** - 验证 Controller + Service + Repository
3. **安全测试** - 权限验证、SQL 注入、XSS 防护
4. **边界测试** - 空值、超长输入、特殊字符

### 测试用例设计

#### 3.1.1 修改密码 API (`PUT /api/v1/user/password`)
| 用例 ID | 场景 | 输入 | 预期结果 |
|---------|------|------|----------|
| PWD-001 | 正常修改 | 正确旧密码 + 合规新密码 | 200 OK，密码更新成功 |
| PWD-002 | 旧密码错误 | 错误旧密码 | 400 Bad Request |
| PWD-003 | 新密码过短 | 新密码<6 位 | 400 Bad Request |
| PWD-004 | 新密码为空 | 空密码 | 400 Bad Request |
| PWD-005 | 未授权访问 | 无 Token | 401 Unauthorized |
| PWD-006 | 新旧密码相同 | 相同密码 | 400 Bad Request |

#### 3.1.2 查询用户资料 API (`GET /api/v1/user/{userId}/profile`)
| 用例 ID | 场景 | 输入 | 预期结果 |
|---------|------|------|----------|
| PROF-001 | 顾问查询用户 | 顾问 Token + 有效 userId | 200 OK，返回完整资料 |
| PROF-002 | 普通用户查询他人 | 用户 Token + 他人 userId | 403 Forbidden |
| PROF-003 | 查询不存在用户 | 无效 userId | 404 Not Found |
| PROF-004 | 未授权访问 | 无 Token | 401 Unauthorized |

#### 3.1.3 任务确认 API (`POST /api/v1/tasks/{taskId}/confirm`)
| 用例 ID | 场景 | 输入 | 预期结果 |
|---------|------|------|----------|
| CONF-001 | 正常确认 | 有效 taskId + 用户 Token | 200 OK，任务状态更新 |
| CONF-002 | 重复确认 | 已确认 taskId | 200 OK（幂等）或 400 |
| CONF-003 | 确认他人任务 | 非本人 taskId | 403 Forbidden |
| CONF-004 | 确认不存在任务 | 无效 taskId | 404 Not Found |
| CONF-005 | 未授权访问 | 无 Token | 401 Unauthorized |

#### 3.1.4 更新资料 API (`PUT /api/v1/user/profile`)
| 用例 ID | 场景 | 输入 | 预期结果 |
|---------|------|------|----------|
| UPD-001 | 正常更新 | 合规资料数据 | 200 OK，资料更新成功 |
| UPD-002 | 必填字段缺失 | 缺少必填字段 | 400 Bad Request |
| UPD-003 | 字段超长 | 超出长度限制 | 400 Bad Request |
| UPD-004 | 更新他人资料 | 他人 userId | 403 Forbidden |
| UPD-005 | 未授权访问 | 无 Token | 401 Unauthorized |

## 🧪 任务 3.3: E2E 测试（使用 Agent Browser）

### 测试场景
使用 Agent Browser 技能执行端到端自动化测试，模拟真实用户操作流程。

### E2E 测试流程

#### 3.3.1 密码修改 E2E
```
1. 打开登录页 → 输入账号密码 → 登录
2. 进入个人中心 → 点击"修改密码"
3. 输入旧密码、新密码、确认密码 → 提交
4. 验证成功提示 → 退出登录
5. 使用新密码重新登录 → 验证成功
```

#### 3.3.2 资料编辑 E2E
```
1. 登录系统
2. 进入个人资料页面
3. 修改背景/目标字段
4. 保存 → 验证成功提示
5. 刷新页面 → 验证数据持久化
```

#### 3.3.3 任务确认 E2E
```
1. 登录系统（用户账号）
2. 进入任务列表
3. 找到待确认任务 → 点击"确认"
4. 验证状态变更为"已确认"
5. 刷新页面 → 验证状态持久化
```

### 输出产物
- `e2e-test-results.md` - E2E 测试执行结果
- `e2e-screenshots/` - 测试过程截图
- `e2e-logs/` - 浏览器控制台日志
