#### API 接口测试（更新：正确路径 `/back-server/api/v1/`）

| 接口 | 预期 | 实际 | 状态 |
|------|------|------|------|
| `GET /back-server/actuator/health` | 200 UP | ✅ 200 `{"status":"UP"}` | ✅ 通过 |
| `POST /back-server/api/v1/auth/login` | 200 + Token | ✅ 200 + JWT Token | ✅ 通过 |
| `GET /back-server/api/v1/tasks` | 200 + 任务列表 | ⚠️ 403 Forbidden | ⚠️ 待修复 |

**验证命令**:
```bash
# 健康检查
curl -s https://plan.shujuyunxiang.com/back-server/actuator/health
# 返回：{"status":"UP","groups":["liveness","readiness"]}

# 登录接口
curl -X POST https://plan.shujuyunxiang.com/back-server/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
# 返回：{"code":200,"message":"登录成功","token":"eyJhbGciOiJIUzI1NiJ9...","expiresIn":86400000}

# 任务列表（带 Token）
curl -s https://plan.shujuyunxiang.com/back-server/api/v1/tasks \
  -H "Authorization: Bearer {token}"
# 返回：403 Forbidden
```

#### 问题根因

⚠️ **JWT Token 验证未配置**

后端分析 (`SecurityConfig.java`):
- ✅ `/api/v1/auth/**` - 允许匿名访问
- ✅ `/actuator/health` - 允许匿名访问
- ⚠️ `/api/v1/tasks/**` - 需要认证，但**未配置 JWT 过滤器**

代码审查发现：
1. `UserService.java` - JWT Token 生成正常
2. `SecurityConfig.java` - 缺少 JWT 认证过滤器
3. `TaskController.java` - 无权限注解，依赖 Spring Security

#### 验收结论

**⚠️ 部分通过** - 后端 API 已部署且可访问，但 JWT 认证链路不完整

**需要修复**:
1. 添加 JWT 认证过滤器 (`JwtAuthenticationFilter`)
2. 配置 SecurityConfig 使用 JWT 过滤器
3. 验证 Token 解析和权限校验
4. 重新执行任务接口测试
