# 职业发展计划 2026 - 任务管理

## 工作流程

1. **任务定义**: 所有任务在 main 分支的 tasks/ 目录定义
2. **领取任务**: Workers 合并 main 分支到自己的 feature 分支
3. **执行任务**: 完成交付物并更新任务 markdown
4. **提交推送**: 推送到自己的 feature 分支
5. **验证**: qa-tester 验证并记录

## 任务状态

- ⏳ 进行中
- ✅ 已完成
- 🚨 需要帮助
- ❌ 已取消

## 优先级

- P0 紧急 - 立即处理
- P1 高 - 今天完成
- P2 中 - 本周完成

## 文件命名

task-worker-name-task-name.md

## 更新流程

完成任务后：
1. 更新任务 markdown 的状态
2. 填写 Git 提交信息
3. 填写完成记录
4. 提交推送到 feature 分支
