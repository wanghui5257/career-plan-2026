### 1.1 画像一：职场新人小李

| 属性 | 描述 |
|------|------|
| **姓名** | 小李 |
| **年龄** | 24 岁 |
| **职业** | 初级软件工程师 |
| **工作年限** | 1 年 |
| **城市** | 北京 |
| **教育背景** | 本科，计算机科学与技术 |

#### 背景故事
小李刚毕业一年，在一家互联网公司做后端开发。他对职业发展感到迷茫，不知道应该深耕技术还是转向管理。每天工作很忙，但没有系统的学习计划，感觉成长缓慢。

- ❌ 工作忙，没时间系统学习
- ❌ 信息过载，不知道学什么
- ❌ 缺乏反馈，不知道进步没有
- ❌ 对职业方向感到困惑

> "下班后想学点东西，但打开各种教程不知道从哪开始。希望有个工具能帮我规划每天学什么，还能看到自己的进步。"

1. 每日学习计划自动推送
2. 学习进度可视化
3. 技能成长曲线
4. 同行人对比参考

## 3. 痛点和机会点总结

### 3.1 核心痛点

| 痛点类别 | 具体表现 | 影响用户 |
|----------|----------|----------|
| **方向迷茫** | 不知道职业发展方向，缺乏清晰目标 | 所有用户 |
| **信息过载** | 学习资源太多，不知道选什么 | 小李、王姐 |
| **时间不足** | 工作忙，没时间系统学习 | 所有用户 |
| **缺乏反馈** | 不知道进步没有，没有成就感 | 小李、阿强 |
| **年龄焦虑** | 担心竞争力不足，害怕被淘汰 | 王姐、阿强 |
| **准备不足** | 简历、面试缺乏系统准备 | 王姐、小李 |

### 3.2 产品机会点

| 机会点 | 解决方案 | 优先级 |
|--------|----------|--------|
| **智能规划** | AI 根据用户情况生成个性化学习/求职计划 | P0 |
| **进度可视化** | 用图表展示学习进度和成长曲线 | P0 |
| **碎片化学习** | 支持每日 15-30 分钟的微学习任务 | P1 |
| **能力评估** | 定期评估用户能力，展示差距 | P1 |
| **社区支持** | 同路人交流，减少焦虑感 | P2 |
| **成功案例** | 展示相似背景用户的成功路径 | P2 |

### 3.3 设计原则

1. **简洁优先** - 避免信息过载，每次只展示必要内容
2. **正向反馈** - 及时展示进步，增强用户信心
3. **灵活适配** - 支持不同职业阶段用户的不同需求
4. **数据驱动** - 用就业市场数据指导用户决策
5. **温暖陪伴** - 减少焦虑，营造支持性氛围
